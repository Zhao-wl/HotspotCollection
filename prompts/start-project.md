# 开始新项目 - 提示词模板

> 使用方法：复制下方提示词到 Cursor 的 Agent 聊天窗口，替换 `[...]` 中的内容。

---

## 提示词

```
我要开始一个新的长期项目。

## 项目描述
[在这里详细描述你的项目，越具体越好。例如：
- 构建一个全栈待办事项 Web 应用，前端用 React，后端用 FastAPI
- 支持用户注册登录、创建/编辑/删除待办事项、设置优先级和截止日期
- 使用 SQLite 数据库，部署到 Docker]

## 技术栈偏好（可选）
[列出你希望使用的技术栈，如果没有特别要求可以删除这一节]

## 其他要求（可选）
[任何其他需求或约束，如果没有可以删除这一节]

---

请按照 long-running-harness 规则进入初始化模式：
1. 分析我的需求，将项目拆分为细粒度的可验证特性
2. 创建 agent-state/features.json 特性列表（按优先级排序）
3. 创建 agent-state/init.ps1 环境初始化脚本
4. 搭建项目基础脚手架
5. 初始化 agent-state/progress.md 进度日志
6. 做好第一次 git commit
```

---

## 提示

- 项目描述越详细，生成的特性列表越准确
- 建议明确列出核心功能和非核心功能
- 如果有参考应用或设计稿，也可以在描述中提及
