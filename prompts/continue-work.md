# 继续开发 - 提示词模板

> 使用方法：每次开始新的 Cursor Agent 会话时，复制下方提示词到聊天窗口。

---

## 提示词（标准版）

```
继续开发。请按照 long-running-harness 规则：
1. 读取 agent-state/progress.md 和 git log 了解当前进度
2. 检查 agent-state/features.json 找到下一个待完成特性
3. 增量实现该特性，完成后验证并提交
```

---

## 提示词（指定特性版）

> 如果你想让 Agent 优先处理某个特定特性，使用这个模板。

```
继续开发。请按照 long-running-harness 规则，优先实现特性 [F00X]：
1. 先读取 agent-state/ 了解当前状态
2. 实现 [F00X] 并按验收标准逐条验证
3. 验证通过后提交并更新进度
```

---

## 提示词（修复 Bug 版）

> 如果你发现了需要修复的问题，使用这个模板。

```
在继续新特性之前，有一个 bug 需要先修复：

## Bug 描述
[描述 bug 的现象、复现步骤]

请按照 long-running-harness 规则：
1. 先读取当前进度了解上下文
2. 修复此 bug 并验证
3. 提交修复后再继续下一个特性
```

---

## 提示

- 每次新会话都建议使用这些提示词，确保 Agent 先恢复上下文
- Agent 会自动读取 progress.md 和 features.json，不需要你手动告诉它项目状态
- 如果 Agent 试图跳过健康检查直接开始编码，提醒它先按照规则恢复上下文
